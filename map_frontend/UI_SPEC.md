# MAP UI/UX Design System (v2.0 — Bloomberg Terminal Doctrine)

> 本文档为 MAP 前端全部页面的**绝对红线**。任何 PR 违反以下任一条款，必须打回重做。
> 适用于人工编码与 AI 代码生成。无例外。无宽限期。

---

## 1. 设计哲学 (Design Philosophy)

本系统的视觉语言对标 Bloomberg Terminal / Reuters Eikon —— 以**信息密度和阅读效率**为唯一度量。

### 1.1 极客与专业 (Geek & Professional)
深蓝灰主色调，克制使用高饱和色彩。**绝对禁止使用装饰性 Emoji、插画、吉祥物或任何非数据驱动的视觉元素。**

### 1.2 数据严谨 (Data Integrity)
数字对齐重于一切。消除视觉歧义。一个像素的错位都是缺陷。

### 1.3 读写分离 (Read/Write Separation)
通过背景深度和阴影，明确区分"只读数据"与"可操作域"。用户必须在不思考的情况下感知哪些区域可交互。

### 1.4 去具象化 / De-iconification（核心原则）
**高级金融终端不需要花哨的具象图形来辅助理解。数据和文字本身就是最好的 UI。**
- 专业的投资决策者不依赖"一个小人图标"来理解"这是用户"、不依赖"一个火箭"来理解"这是增长"。
- 信息传达必须通过**文字、数字、颜色和空间关系**完成。
- 凡是试图用图形"让界面更友好"的做法，在此系统中一律视为视觉噪音。
- 唯一允许的非文字视觉元素：**抽象几何竖线**（见第 3 节）。

## 2. 字体与排版规范 (Typography & Legibility)

为保证高管及业务人员的阅读体验，提升全局基础字号。

### 2.1 字号红线
| 级别 | Tailwind 类 | 像素 | 用途 |
|------|-------------|------|------|
| 页面标题 | `text-lg` | 18px | 页面唯一主标题 |
| 栏目标题 | `text-base` | 16px | 主要卡片/面板标题 |
| 子栏目标题 | `text-sm` | 14px | 子栏目/区块标题 |
| 正文/数据 | `text-xs` | 12px | **表格数据、表单标签、正文的最小常规字号** |
| 辅助说明 | `text-[11px]` | 11px | 仅用于极次要的补充（单位、图例） |

**禁令：**
- **严禁 `8px`、`9px`、`10px`**。无例外。无"但是"。
- **严禁 `text-[11px]` 用于任何需要用户主动阅读的正文内容。**
- `text-[11px]` 仅限：单位后缀、图例文字、时间戳标注。

### 2.2 数字排版
- 所有金额、比例、收益率数据，必须使用 `font-mono tabular-nums`。
- 上下行数字必须绝对对齐。比例符号 `%`、货币符号 `¥`/$` 对齐方式必须统一。

## 3. 去图标化与几何装饰规范 (Iconography Ban & Vertical Bar Doctrine)

> 本节为 v2.0 核心变更。原有代码中残留的图标库引用视为技术债，须逐步清除。

### 3.1 图标库禁令

**严禁使用任何第三方图标库，包括但不限于：**
- `@element-plus/icons-vue`
- `lucide-vue-next`
- `@heroicons/vue`
- `@iconify/vue`
- 任何 SVG sprite / icon font

**理由：** 具象图标（人形、火箭、文件夹、齿轮）在 Bloomberg 级终端中是视觉噪音。专业用户通过文字和位置识别功能，不需要图腾。

### 3.2 唯一合法视觉元素：抽象几何竖线 (Vertical Bar)

所有标题强调、状态提示、警告标志，统一使用**竖向矩形色条**。
通过改变色条的 `background-color` 表达不同语义，不使用任何图形。

#### 标准竖线 HTML（Tailwind）

```html
<!-- 标准标题竖线（品牌蓝） -->
<div class="w-1.5 h-4 bg-[#3B9EFF] rounded-sm shrink-0"></div>

<!-- 看多/上涨状态竖线（金融红） -->
<div class="w-1.5 h-4 bg-[#F04864] rounded-sm shrink-0"></div>

<!-- 看空/下跌状态竖线（金融绿） -->
<div class="w-1.5 h-4 bg-[#00C9A7] rounded-sm shrink-0"></div>

<!-- 预警/风险竖线（使用品牌蓝的低透明度变体表达谨慎，而非黄色） -->
<div class="w-1.5 h-4 bg-[#3B9EFF]/40 rounded-sm shrink-0"></div>
```

#### 竖线色值映射表

| 语义 | 色值 | Tailwind |
|------|------|----------|
| 默认/标题/中性/提示 | `#3B9EFF` | `bg-[#3B9EFF]` |
| 看多/上涨/高配/利好 | `#F04864` | `bg-[#F04864]` |
| 看空/下跌/低配/利空 | `#00C9A7` | `bg-[#00C9A7]` |
| 弱调/辅助/次要 | `#3B9EFF/40` | `bg-[#3B9EFF]/40` |
| 禁用/无效/占位 | `#64748B` | `bg-[#64748B]` |

### 3.3 状态表达的合法方式

| 场景 | 合法做法 | 违规做法 |
|------|---------|---------|
| 标题左侧强调 | 蓝色竖线 `div` | 使用图标组件 |
| 表格行内状态 | 彩色文字 + 圆角 `span` | 使用 `Check`/`Close` 图标 |
| 警告提示 | 红色或蓝色竖线 + 文字 | 使用 `Warning` 三角形图标 |
| 按钮内标识 | 纯文字，必要时加竖线 | 使用 `Plus`/`Arrow` 图标 |
| 成功状态 | 绿色文字或竖线 | 使用 `Check` 打勾图标 |
| 删除/危险操作 | 红色文字 | 使用 `Trash`/`Delete` 图标 |

### 3.4 例外条款
以下场景暂时允许保留图标，但必须列入技术债清单并逐步替换：
- 录音控件（播放/暂停/停止）—— 功能性按钮，后续用文字标签替代
- 文件类型标识（PDF/Excel）—— 后续用纯文字标签替代

## 4. 色彩规范 (Color Palette — Converged)

> v2.0 核心变更：**大幅收敛可用色彩**。整个系统只允许三类颜色。

### 4.1 背景层级（严禁修改）

| 层级 | 色值 | 用途 |
|------|------|------|
| Base | `#161922` | 页面底色 |
| Panel | `#1A1E2B` | 卡片、区块底色 |
| Elevated | `#202431` | 悬浮面板、弹窗、Header |

### 4.2 文本层级（严禁修改）

| 层级 | 色值 | 用途 |
|------|------|------|
| Title | `#FFFFFF` | 页面/栏目标题 |
| Primary | `#E8ECF4` | 正文、表头 |
| Secondary | `#B4BAC9` | 表格内容、描述 |
| Muted | `#94A3B8` | 辅助说明、时间戳 |
| Disabled | `#64748B` | 禁用、占位 |

### 4.3 品牌色（唯一的高亮中性色）

**品牌蓝 `#3B9EFF` 是全系统唯一允许使用的非金融语义高亮色。**

用途包括但不限于：
- 标题竖线、选中态、按钮主色、链接色
- "中性"观点标签、均值标记、默认高亮
- 分类标签、分组标识、状态指示

**透明度变体合法使用：**
- `bg-[#3B9EFF]/10` — 浅底色
- `bg-[#3B9EFF]/20` — 中底色
- `border-[#3B9EFF]/30` — 边框
- `text-[#3B9EFF]` — 文字高亮

### 4.4 金融语义色（唯二允许的高饱和色）

**"红涨绿跌"是不可违反的绝对语义，沿用 A 股习惯。**

| 语义 | 色值 | Tailwind |
|------|------|----------|
| 看多/上涨/盈利/高配/利好 | `#F04864` | `text-[#F04864]` `bg-[#F04864]` |
| 看空/下跌/亏损/低配/利空 | `#00C9A7` | `text-[#00C9A7]` `bg-[#00C9A7]` |

### 4.5 色彩禁令

**以下颜色严禁在 MAP 系统中使用（除非代码评审明确批准）：**

| 禁止色值 | 原用途 | 替代方案 |
|----------|--------|---------|
| `#FFAB00` (Amber/黄) | 预警/中性偏乐观 | 使用 `#3B9EFF` 品牌蓝表达中性 |
| `#22D3EE` (Cyan/青) | 中性偏谨慎/辅助 | 使用 `#3B9EFF` 品牌蓝 |
| `#8B5CF6` (Purple/紫) | 委员角色标识 | 使用 `#3B9EFF` 不同透明度区分角色 |
| `#F97316` (Orange/橙) | 提醒/催办 | 使用 `#3B9EFF` 或 `#F04864`（看严重程度） |
| `#FF9500` (Amber-橙) | 风险标记 | 使用 `#F04864` 红色（风险=看空语义） |
| `#FF3B30` (Red-bright) | 录音/停止 | 使用 `#F04864` 标准红 |

**原则：如果它不是"看多"（红）也不是"看空"（绿），它就必须是品牌蓝或灰度。**

## 5. 表格与网格规范 (The "DongZong" Grid Rule)

针对所有包含对比性质的财务、收益率表格，严格执行以下红线：

### 5.1 固定列宽
强制开启 `table-fixed`。所有数据对比列必须使用统一的绝对宽度（如 `w-24` 或 `w-[100px]`）。

### 5.2 绝对右对齐
所有金额、百分比数值必须绝对右对齐 `text-right`。

### 5.3 表头释义
对于"近1月"、"YTD"等时间区间表头，必须配有文字标记 `(i)`，Tooltip 需明确标示计算的自然日区间（如 `2026-03-15 至 2026-04-15`）。**禁止使用问号图标，使用纯文字 `(i)` 即可。**

### 5.4 行间距
数据行高不低于 `py-2.5`，确保在投委会大屏上的可读性。

## 6. 交互与术语规范 (Interaction & Terminology)

### 6.1 SASA 命名法
全局统一层级术语：`委员会指引`、`部门指引`、`产品指引`、`实际持仓`。

### 6.2 扁平化导出
所有树形折叠表格（TreeGrid）上方必须提供 `[导出数据 (Excel)]` 按钮。导出格式为 2D 扁平横表。**禁止使用任何图标装饰导出按钮。**

### 6.3 区域响应 (Zones)
- `.readonly-zone`: 下沉阴影，无 hover 效果。
- `.operable-zone`: 略浅的底色，hover 时边框亮起（`hover:border-[#3B9EFF]/50`）。

## 7. 视觉元素规范

### 7.1 Emoji 禁令
**严禁出现任何 Emoji。** 包括但不限于 Unicode emoji（📚📊⚖️📝）、CSS emoji、图片形式的 emoji。

### 7.2 标题装饰
所有级别标题左侧必须统一使用竖线修饰，**严禁使用图标**。
- 一级标题: `w-1 h-5 bg-[#3B9EFF] mr-3 rounded-sm`
- 二级/三级标题: `w-1.5 h-4 bg-[#3B9EFF] rounded-sm shrink-0`

### 7.3 按钮规范
- 按钮内只允许纯文字。禁止嵌入图标组件。
- 主按钮: `bg-[#3B9EFF] text-white` 或渐变 `bg-gradient-to-r from-[#3B9EFF] to-[#3B9EFF]/80`
- 危险按钮: `border-[#F04864]/30 text-[#F04864]`
- 次要按钮: `border-[#3B9EFF]/25 text-[#3B9EFF]`

### 7.4 空间与留白
- 卡片间距统一 `gap-4` 或 `space-y-4`
- 卡片内边距统一 `p-5` 或 `px-5 py-3.5`
- 圆角统一 `rounded-xl`（卡片）或 `rounded-lg`（内部元素）
- **严禁 `rounded-full` 用于任何非状态圆点的元素。**

## 8. 合规检查清单 (Pre-PR Checklist)

每个 PR 合并前，提交者必须确认以下条款：

- [ ] **无 `8px`/`9px`/`10px` 字号**（全文搜索 `text-[8px]` `text-[9px]` `text-[10px]`）
- [ ] **无第三方图标组件**（搜索 `icons-vue`、`lucide`、`heroicons`、`iconify` 的 import 语句）
- [ ] **无 Emoji**（搜索常见的 emoji Unicode 范围或视觉检查）
- [ ] **无禁用色彩**（搜索 `#FFAB00`、`#22D3EE`、`#8B5CF6`、`#F97316`、`#FF9500`、`#FF3B30`）
- [ ] **所有数字使用 `font-mono tabular-nums`**
- [ ] **所有金额/百分比右对齐 `text-right`**
- [ ] **表格开启 `table-fixed`**
- [ ] **标题竖线使用 `div` 而非图标组件**
