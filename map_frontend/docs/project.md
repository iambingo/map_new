## 5. 已知风险与决策

| # | 风险/问题 | 影响 | 处理方案 |
|---|---|---|---|
| R-01 | shadcn-vue 官方仅支持 Vue 3，Vue 2.7 兼容性存疑 | 高 | 开发初期验证兼容性；必要时切换 Element Plus 或升级 Vue 3 |
| R-02 | Pinia 3 可能移除 Vue 2 支持 | 中 | 评估 `@pinia/vue2` 插件可用性，备选 Vuex 4 |
| R-03 | @tanstack/vue-query v5 仅支持 Vue 3 | 中 | 使用 v4（兼容 Vue 2.7）或降级为手动 Axios 管理 |
| R-04 | PostgreSQL 金融时序数据写入性能 | 低 | 先用 JSONB + 分区表，后期评估 TimescaleDB |
| R-05 | 原型使用 React + Recharts，与技术栈（Vue + ECharts）不一致 | 中 | 仅参考原型业务逻辑，正式实现使用 Vue；图表用 ECharts 替换 Recharts |
| R-06 | 模型中心 Python 执行环境隔离 | 高 | 后端需提供沙箱执行（如 Docker + subprocess），禁止直接执行未审计代码 |
| R-07 | Wind 数据接口授权与并发限制 | 中 | 后端统一接入 Wind，添加限流与缓存层，避免客户端直连 |
| R-08 | 调仓指令生成的审批流设计 | 中 | 需明确审批节点（PM → 风控 → 部门负责人），建议 Phase 2 引入工作流引擎 |
